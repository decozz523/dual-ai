# Оценка проекта dual-ai

## Итоговая оценка
**7/10 (сильный MVP, готовый к демонстрации, но требует усиления продакшен-процессов).**

## Что уже хорошо
- Чёткий продуктовый фокус: дуо-чат с разными ролями ботов (Samii и Vivi), понятный value proposition прямо на первом экране.
- Богатый UX для MVP: быстрые сценарии старта, режимы диалога, шаблоны, экспорт, управление диалогами.
- Локализация RU/EN и аккуратная работа с UI-состояниями (темы, модальные окна, настройки, профили, фильтры).
- Вынесен серверный прокси для OpenRouter (`api/chat.js`) — это правильнее, чем прямые вызовы LLM API из браузера.
- Есть задел под монетизацию (планы Free/Plus/Pro, Telegram-бот, коды активации, Supabase).

## Ключевые риски и узкие места
1. **Поддерживаемость фронтенда**  
   `app.js` очень крупный и объединяет сразу много зон ответственности (UI, состояние, i18n, бизнес-логика, авторизация, монетизация). Это усложнит развитие и отладку.

2. **Минимальная эксплуатационная документация**  
   `README.md` пока почти пустой, отсутствуют инструкции запуска, переменные окружения, деплой и troubleshooting.

3. **Безопасность и операционная зрелость**
   - В `api/chat.js` нет ограничений по размеру/частоте запросов и нет явной валидации payload.
   - В `bot.py` используются прямые константы для секретов (пусть и заглушки), но лучше сразу стандартизировать через env-переменные и `.env.example`.

4. **Тестирование и CI**  
   Не видно unit/integration/e2e тестов и автоматических quality gates (lint/test/build в CI).

## Приоритетный план улучшений
### P0 (сделать в первую очередь)
- Добавить полноценный `README.md`: запуск локально, env-переменные, архитектура, деплой.
- Разбить `app.js` на модули: `state`, `i18n`, `ui`, `api`, `dialogs`, `billing`.
- В `api/chat.js` добавить валидацию входа, ограничение размера запроса и простейший rate limiting.
- Перевести `bot.py` на конфиг через переменные окружения + добавить `.env.example`.

### P1
- Настроить линтеры и форматтеры (ESLint/Prettier для JS, Ruff/Black для Python).
- Добавить smoke-тесты для ключевых пользовательских сценариев (отправка сообщения, смена языка, активация кода).
- Включить CI pipeline (например, GitHub Actions) с обязательными проверками.

### P2
- Добавить телеметрию/аналитику событий (вход, активация кода, ошибки API, конверсия в апгрейд).
- Вынести тексты i18n в отдельные JSON-файлы для масштабирования локализаций.

## Продуктовая перспектива
Проект выглядит как **живой и уже полезный MVP** с хорошим потенциалом удержания за счёт персонажей и сценарного UX. Следующий шаг к росту — не столько новые фичи, сколько **стабилизация инженерной базы**: документация, модульность, безопасность API и автоматизированные проверки.
